Project Requirements

Functional Requirements

Module structure: Creation and implementation
Modules creation and implementation will take place during the beginning phases of programming the application. Modules will have lesson that will give the app user the module information and game will correlate to the module.
Data: Storage and analysis of data
Data of the user's engagment will be stored in our cloud preference, Google Firebase. Data that will be stored in Google Firebase will be the user's module and game completion. This data will be able to be viewable in the administor web portal.
Gamification: Visual aesthetic
Game that will be uploaded to the module will need to be interesting and informative to the application user's. The game will take a pixelated aesthetic.
User: Target audience and Ease of Use
Ease of use is a main priority for application user's since the target audience is parents of 0-3 years old.

Non-Functional Requirements

Usability
Usability means that the app User Interface will need to be as straight forward as possible will still be engaging. The app will need to be accessible to underprivilage parent with lower reading capabilities as well.
Game Response
Game will be measured by the way the application user interacts with the module's games. The testing of the game will take into account the time and completion of the game. Games should take less than 3 minutes to complete and most of the games should be completed in no less than 3 times. x
For this project we are following the Requirements Specification Document that we have written as a group with the input of our sponsor. This document goes into more details of our project's requirements - both functional and non-functional - and the expectations to be fulfilled by our team.
